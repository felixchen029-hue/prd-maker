# 前置访谈与 brief

## 目标

用尽量少的提问确定文档结构、分析范围和证据策略。问答的结果必须落到 `prd-brief.yaml`，不能只停留在聊天记录中。

## 两轮访谈

第一轮最多询问三个高影响问题：

1. 产物是 PRD、个人总结还是管理汇报？主要读者要做什么决定？
2. 本次聚焦哪个 IaaS 产品、实例类型或功能，明确不处理什么？
3. 用户已有材料有哪些，是否启用联网搜索和 llm-wiki？

第二轮仅追问启用模块：

- 市场调研：地区、时间、客户类型、市场口径。
- 竞品分析：候选竞品、产品版本、对比维度、是否比较价格。
- 实例功能：生命周期、规格、镜像、网络、存储、计费、配额和可观测性。
- 总结：周期、职责、目标、结果、阻塞和下一步。
- 汇报：决策事项、备选方案、资源投入、成本和风险。

## 框架确认

展示以下内容后再写正文：

- 文档目录和章节顺序。
- 每一章节的 `skip / brief / standard / deep` 深度。
- 调研来源、引用要求、时间和地区边界。
- 计划生成的图表。
- 已知假设、材料缺口和不会覆盖的内容。

## 默认值

用户要求直接执行时采用：

- `document_type: prd`
- `industry: cloud-computing`
- `focus: iaas-compute`
- 联网搜索开启，llm-wiki 为 `auto`
- 引用必须保留
- 产物为 Markdown + SVG
- 市场和竞品分析为 `standard`
- 需求、方案和验收为 `deep`
